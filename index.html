Getting Started

YObject is an engine that extends Geometry Dash's creation capabilities. It allows you to write custom code that runs alongside your levels, creating complex behaviors, and expanded functionality beyond traditional triggers.

Why YObject?

Beyond Triggers: Write actual code instead of relying solely on trigger combinations.

Object-Oriented: Create reusable object structs and traits with inheritance and encapsulation(will be soon).

Geode Integration: Seamlessly works with the Geode mod loader ecosystem.

Variables and Values

Since you can't really print Hello, world! to the console using Geometry Dash triggers, we're going to get straight into the main features of YObject, the first of which being variables.

In YObject, variables are defined like this:

Code 
// [variable name] = [value]
a = 10
b = 60
c = a + b

// or

// const [variable name] = [value] (to make them constant)
const d = 10
const e = 60
const f = d + e

YObject Values

So, what kind of stuff can you put in your variables? Well, a lot of different things. Some value types are ones you might be familiar with from other languages:

Code 
num = 10

text = "Hello, I'm a string!"

array = [1, 2, 3]

boolean = true

These values can be used as you would in any other programming language. Here are some examples:

Code 
num = -(1000 * 1000) / 10 // -100000

name = "YaY"
text = "Hello, my name is " + name + "!"

array = [1, 2, 3]
my_number = array[2] // 3

array += [4] // after this, the array will now be [1, 2, 3, 4]

boolean = true
boolean2 = !boolean // false

IDs

In YObject, there are also values that that are useful specifically when making Geometry dash levels:

Code 
groupID = 10g
colorID = 10c
itemID  = 10i
timerID  = 10t
blockID = 10b

// And get next free id:

groupID = ?g
colorID = ?c
itemID  = ?i
timerID  = ?t
blockID = ?b

Basic Triggers

As you probably know, to move objects in Geometry Dash, you move the group the objects are in. If you wanted to, say, move group 5 ten units on the x-axis in 0.5 seconds, you would add a move trigger with those settings. In YObject we do it like this:

Code 
// group.move(x, y, [duration], [easing type] = 0)
5g.move(10, 0, 0.5)

You can add delay to your triggers using wait()

Code 
wait(1.5)
1g.move(10, 0, 1)
wait(1)
2g.move(20, 0, 1)

Conditions

YObject has a quite standard syntax for if-statements:

Code 
if condition {
    // code
}

if condition {
    // code
} else {
    // code
}

if condition {
    // code
} else if other_condition {
    // code
} else {
    // code
}

Also if you have item or timer in condition it will create item compare trigger to make in-game branching

Code 
if 10i == 5 {
    // it will create compare trigger
}

Functions

YObject has 2 types of functions (and one of them has 2 subtypes):

Code 
// define a function:
fn function(a: int, b: float = 10.5) {
    1g.move(a, 0, b)
}

// simple call:
func(10, 15)

// trigger call with remembering group
// (it will create spawn trigger and spawn this
// triggers with remembering their group):
func(10, 15)!

// trigger call without remembering group with new group
func(10,15)!!





// also you can return value from function:

fn function2(a: int, b: float = 10.5) -> float {
    return a + b
}


// and you can overload functions:

fn function2(a: int, b: float, c: int) -> float {
    return a + b - c
}

Structures

You can define structures in YObject:

Code 
struct MyStruct {
    intProperty: int
    floatProperty: float
}

You can create and interact with structs like this:

Code 
myObject = MyStruct

myObject.intProperty = 10
myObject.floatProperty = 20.3

Implementations

You can implement structures in YObject:

Code 
impl MyStruct {
    fn staticFunc(a: int) -> int {
        return a + 2
    }
    fn nonstaticFunc(self, a: int) -> int {
        return self.intProperty + a
    }                         
}


MyStruct.staticFunc(10) // return 12

a = MyStruct
a.intProperty = 5
a.nonStaticFunc(10) // return 15 because 5 + 10 = 15

Traits

You can define traits in YObject:

Code 
trait MyTrait {
    fn func(self) -> int
    fn func2(self) -> float
    { 
       // you don't need to write code of this function
       // but if you do that, it will be default implementation
    }
}

You can implement traits for structs:

Code 
impl MyTrait for MyStruct {
    fn func(self) -> int {
        return self.intProperty
    }                        
}


myObj = MyStruct

myObjButTrait = (myObj)MyTrait // casting will be shown soon
myObjButTrait.func() // return myObj intProperty

Casts

You can cast variables in YObject:

Code 
integer = 1
// you can cast like this:
floatNumber = (integer)float

// or this:
floatNumber = 1float

// [number][type] or (variable)[type]

Libraries

You can import other files in YObject:

Code 
import "root/filename.y" as libraryName

libraryName.function() // some function from this file

Also you can extract all from this file:

Code 
extract import "root/filename.y"

function() // some function from this file,
// you can use it without calling name of this library
